  • Controls

    Right mouse button to pan.
    Left mouse button to use a tool.
    Hold the backtick key or middle mouse button to open the tool selection menu
    Release the mouse while hovering over the desired tool to select it, or press the arrow keys. Tools are:
    Drag tool - drag things around. Inspect tool - click on a creature to see information about it. Food tool - click to place food. Editor tool - add creatures with custom properties.


  • Help! I found a bug!

    If possible, get a screenshot of it, and send it to me with a detailed description of how you triggered it or what you were doing when it happened.


  • Command information

    You can press the slash key to show and hide a command box. Within this box, you can toggle debug-only preferences:
    "visionCone" (shows what creatures can see) "hearingRange" (shows what creatures can hear) "senseArea" (shows box being checked for other creatures) "hitboxGrid" (shows a representation of the hitboxes) "showId" (shows creature IDs as floating text) "showState" (shows creature states) "drawPath" (shows what paths creatures are following) "drawRelations" (draws lines between creatures representing their attitudes to each other)
    Example usage:
    /showId
    drawPath
    Note that these are case-sensitive. The slash doesn't matter; you can remove it.

  • Code information

    This is going to be long.
    This was written in TypeScript in Visual Studio Code. It uses the Electron framework. Version numbers are my own system, intended merely to convey the approximate number of bugs fixed/features added between each release.
    The canvas is the HTML5 canvas; I could've used WebGL but chose not to in favour of simplicity.

  • Maths information

    This is completely procedurally generated, so there was a lot of maths involved. The biggest equations were the movement and the inverse kinematics systems.
